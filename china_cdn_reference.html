中国大陆常用CDN资源引用指南

重要提示

本文档提供国内常用CDN的相关信息，便于前端开发中引用各类库和框架。由于网络环境差异，建议在实际项目中使用多CDN源备选策略，确保资源加载可靠性。

常用库引用示例

ECharts 图表库（以5.4.3版本为例）

方式1：使用 BootCDN

<script src="https://cdn.bootcdn.net/ajax/libs/echarts/5.4.3/echarts.min.js"></script>

方式2：使用 Staticfile CDN

<script src="https://cdn.staticfile.org/echarts/5.4.3/echarts.min.js"></script>

方式3：使用 jsdelivr

<script src="https://cdn.jsdelivr.net/npm/echarts@5.4.3/dist/echarts.min.js"></script>

方式4：使用 字节跳动 CDN

<script src="https://lf9-cdn-tos.bytecdntp.com/cdn/expire-1-M/echarts/5.4.3/echarts.min.js"></script>

Font Awesome 图标库（以6.4.0版本为例）

方式1：使用 BootCDN

<link rel="stylesheet" href="https://cdn.bootcdn.net/ajax/libs/font-awesome/6.4.0/css/all.min.css">

方式2：使用 jsDelivr

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/@fortawesome/fontawesome-free@6.4.0/css/all.min.css">

多CDN源加载策略（推荐）

为提高资源加载成功率，建议使用多CDN源作为备选方案，下面是一个实用例子：

<!-- 主要CDN源 --> <script src="https://cdn.bootcdn.net/ajax/libs/echarts/5.4.3/echarts.min.js"></script> <!-- 备选CDN源自动加载脚本（在主要CDN加载失败时使用） --> <script> window.addEventListener('error', function(e) { // 只处理脚本加载错误 if (e.target.tagName === 'SCRIPT' && e.target.src.includes('bootcdn')) { console.log('主要CDN加载失败，尝试备用CDN'); // 如果bootcdn加载失败，先尝试staticfile var script1 = document.createElement('script'); script1.src = "https://cdn.staticfile.org/echarts/5.4.3/echarts.min.js"; script1.onerror = function() { // 如果staticfile也失败，再尝试jsdelivr var script2 = document.createElement('script'); script2.src = "https://cdn.jsdelivr.net/npm/echarts@5.4.3/dist/echarts.min.js"; document.head.appendChild(script2); }; document.head.appendChild(script1); } }, true); </script>

提示

在选择CDN时，请考虑以下因素：

  • 目标用户的地理位置和网络环境
  • CDN的稳定性和更新频率
  • 是否支持HTTPS
  • 资源版本的完整性
  • 有无SRI（子资源完整性）支持
